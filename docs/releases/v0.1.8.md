# Token Shredder v0.1.8

A tiny local desktop pet that shreds your AI token spending in real time.

This update focuses on removing the manual verification burden. Token Shredder now has an automated desktop smoke test that launches the production Electron app with an isolated temporary profile, verifies the local collector, then shuts the app down automatically.

## Highlights

- Added `npm run smoke:desktop`.
- Added `npm run release:check:full`.
- Smoke test launches production Electron without a Vite dev server.
- Smoke test uses an isolated temporary `userData` directory.
- Smoke test uses a high temporary local port range to avoid the normal app.
- Automated checks include:
  - `GET /health`
  - native `POST /usage`
  - OpenAI-style `POST /usage`
  - cached token subtraction for OpenAI-style usage
  - disabled-provider guard for `/v1/chat/completions`
  - `DELETE /usage`
- Main process now supports test-only environment overrides for usage port and user data path.
- Updated README, Chinese docs, release guide, roadmap, and launch page copy.

## How To Verify

```bash
npm run release:check:full
```

For packaging:

```bash
npm run dist:mac
npm run release:github
```

## Privacy Notes

- Smoke tests use temporary local app data.
- Smoke tests send local usage numbers only.
- No prompt, completion, message, API key, or Authorization header is required.
- The normal app still listens on `127.0.0.1` by default.

## Known Limits

- macOS is still the primary target.
- Builds are unsigned and not notarized.
- Windows and Linux need community testing.
- The smoke test validates the local API and production startup, not every visual animation frame.
- Full streaming usage extraction is not implemented.
