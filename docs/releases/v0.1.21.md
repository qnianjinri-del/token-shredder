# Token Shredder v0.1.21

A tiny local desktop pet that shreds your AI token spending in real time.

This release focuses on the first five minutes. A newcomer should no longer need to understand every backstage tab before trying the app or connecting real usage.

## Highlights

- Added a first-screen `新手向导` panel in the backstage window.
- The guide separates four real user paths:
  - no-key quick demo
  - provider API Key / local proxy setup
  - direct `POST /usage`
  - Codex local `token_count` monitoring
- Each path shows necessary information, missing fields, next steps, and privacy boundaries.
- Added copyable path checklists and POST examples from the guide.
- Added setup-path tests so direct usage and quick demo paths do not accidentally require API keys.
- Updated README, Chinese getting-started docs, roadmap, release guide, launch kit, and launch page copy.

## How to Try It

Open Token Shredder, right-click the desktop pet, choose `进入后台`, then start at `新手向导`.

Recommended first actions:

1. Click `一键试玩` if you only want to see the pet move.
2. Choose `我有 API Key` if you want the OpenAI-compatible local proxy.
3. Choose `我的项目已有 usage` if your script or agent can report token counts.
4. Choose `我想看 Codex 消耗` if you want to inspect local Codex token_count monitoring.

## Privacy Notes

- The quick demo does not call any provider.
- Direct usage reporting does not require API keys.
- Codex monitoring reads token_count-related numbers, not prompt or completion content.
- Provider proxy copy continues to state that prompt, completion, messages, API keys, and Authorization headers should not be written to usage logs.

## Known Limits

- macOS builds remain unsigned and not notarized.
- Codex monitoring depends on local Codex session files and is not an official billing dashboard.
- Full streaming usage extraction remains limited by provider behavior.
