# Token Shredder v0.1.7

A tiny local desktop pet that shreds your AI token spending in real time.

This update focuses on making Token Shredder more useful after the pet has received real usage: you can now export session data, copy clearer integration recipes, and publish releases with a project script instead of ad-hoc commands.

## Highlights

- Added `Session 导出`.
- Export formats:
  - JSON
  - CSV
  - Markdown
- Session exports include local cost summaries and usage event rows.
- Session exports intentionally exclude prompts, completions, messages, API keys, and Authorization headers.
- Added `接入配方` panel with copy-ready:
  - curl
  - JavaScript fetch
  - Python requests
  - OpenAI SDK local proxy
  - Agent instruction prompt
- Advanced monitoring examples now reuse the same integration example builder.
- Added `npm run release:github` for GitHub Release publishing.
- Added tests for session export helpers.
- Updated README, Chinese docs, roadmap, release guide, and launch page copy.

## How To Try It

1. Open Token Shredder.
2. Right-click the pet and choose `进入后台`.
3. Send a test usage event or use a real integration recipe.
4. Open `Session 导出`.
5. Copy or download JSON, CSV, or Markdown.

## Local API Example

```bash
curl -X POST http://127.0.0.1:17391/usage \
  -H "Content-Type: application/json" \
  -d '{"source":"my-agent","scenarioName":"first real shred","inputTokens":120000,"outputTokens":45000,"cachedInputTokens":30000,"reasoningTokens":8000}'
```

## Privacy Notes

- Session exports contain usage numbers and local cost estimates only.
- Session exports do not include prompts, completions, messages, API keys, or Authorization headers.
- The app remains local-first and listens on `127.0.0.1` by default.
- Prices are user-configured estimates, not official live provider pricing.

## Known Limits

- macOS is still the primary target.
- Builds are unsigned and not notarized.
- Windows and Linux need community testing.
- Full streaming usage extraction is not implemented.
- SDK / CLI wrappers are not implemented yet.
