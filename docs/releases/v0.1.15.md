# Token Shredder v0.1.15

A tiny local desktop pet that shreds your AI token spending in real time.

This update focuses on the first real usage event. The app now does more to explain what a new user needs, how to choose an integration path, and how to hand the setup work to a coding agent without weakening the local-first privacy boundary.

## Highlights

- Added common OpenAI-compatible provider templates, clearly marked as editable examples.
- Added clearer provider setup copy explaining what is required for local proxy mode and what is not required for direct `POST /usage`.
- Added provider setup checklists inside the backstage UI.
- Added a copyable Codex / ChatGPT implementation prompt that asks a coding agent to wire usage reporting into an existing project.
- Added provider field checklist and AI implementation prompt to the integration recipes panel.
- Added Chinese and English Getting Started guides.
- Updated README, Chinese docs, roadmap, launch kit, release guide, and launch page.
- Added tests for provider templates and the new AI implementation prompt.

## How To Try It

1. Open Token Shredder.
2. Right-click the desktop pet and choose `进入后台`.
3. Start from the `开始` tab.
4. Click `运行自动体检`.
5. If you do not know how to connect your project, open `接入` and click `复制给 Codex/ChatGPT`.

## Privacy Notes

- No cloud backend or analytics were added.
- The new implementation prompt explicitly tells coding agents not to send prompts, completions, messages, request bodies, API keys, or Authorization headers to Token Shredder.
- Provider templates are editable examples, not official live provider configuration or pricing.
- Usage collection remains local-first on `127.0.0.1`.

## Known Limits

- macOS remains the primary target.
- Builds are unsigned and not notarized.
- Provider templates may need user adjustment based on the actual provider account, region, and model endpoint.
- The local proxy remains basic.
- Streaming usage extraction remains limited.
- Windows / Linux still need community testing.
