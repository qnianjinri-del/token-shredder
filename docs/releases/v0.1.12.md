# Token Shredder v0.1.12

A tiny local desktop pet that shreds your AI token spending in real time.

This update continues the first-run work. The goal is simple: a new user should be able to open the backstage window, copy one useful integration package, and know exactly what is still missing before trying the local proxy.

## Highlights

- Added a copyable `当前接入包`.
- The integration package includes:
  - setup-readiness summary
  - actual local usage endpoint
  - actual local proxy baseURL
  - model / endpoint ID
  - curl example
  - JavaScript fetch example
  - Python requests example
  - OpenAI SDK proxy example
- `下一步建议` now recommends provider testing when API Key, Base URL, and model ID are already filled, even before the proxy has been enabled.
- After a no-key demo, the card can run a collector test directly.
- Provider setup now validates required fields before saving or testing.
- Provider setup copy now explains that model / endpoint IDs must come from the user's provider and are not official live model lists.
- Added tests for setup-readiness behavior and integration package text.

## How To Try It

1. Open Token Shredder.
2. Right-click the desktop pet and choose `进入后台`.
3. Read `下一步建议`.
4. Click `复制当前接入包`.
5. Paste it into your agent, script, or issue report.

## Privacy Notes

- The copied integration package contains endpoints, examples, and readiness status.
- It does not include API keys.
- It does not include prompts, completions, messages, or Authorization headers.
- The app remains local-first and does not add analytics.

## Known Limits

- macOS remains the primary target.
- Builds are unsigned and not notarized.
- The local proxy remains basic.
- Streaming usage extraction is still not a reliable advertised path.
