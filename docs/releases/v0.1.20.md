# Token Shredder v0.1.20

A tiny local desktop pet that shreds your AI token spending in real time.

This release focuses on distribution quality. The app behavior is intentionally unchanged; the main improvement is that macOS builds no longer ship frontend build-time dependencies as runtime `node_modules` inside the packaged app.

## Highlights

- Reduced macOS package size by moving renderer-only libraries to build-time dependencies.
- Kept the packaged app focused on built `dist/`, `dist-electron/`, `package.json`, and `LICENSE`.
- Added release checklist coverage for packaged `app.asar` contents and artifact size.
- Rebuilt unsigned macOS `.dmg` / `.zip` artifacts for v0.1.20.

## Package Size

Compared with v0.1.19 on the local arm64 build:

- `.dmg`: about 130 MB -> 115 MB.
- `.zip`: about 145 MB -> 129 MB.
- unpacked `.app`: about 326 MB -> 265 MB.

The app still includes Electron, so it is not tiny, but the packaged `app.asar` no longer includes frontend build-time `node_modules`.

## How to Try It

Download the latest macOS build from GitHub Releases, or run from source:

```bash
npm install
npm run dev:desktop
```

For local packaging:

```bash
npm run release:check:full
npm run dist:mac
npm run smoke:package
```

## Local API Examples

```bash
curl http://127.0.0.1:17391/health

curl -X POST http://127.0.0.1:17391/usage \
  -H "Content-Type: application/json" \
  -d '{"source":"release-smoke","scenarioName":"v0.1.20 smoke","inputTokens":120000,"outputTokens":45000}'
```

## Privacy Notes

- Token Shredder remains local-first.
- The collector listens on `127.0.0.1`.
- It records usage counts, source names, timestamps, and local cost estimates.
- It does not upload prompt, completion, messages, API keys, or Authorization headers.

## Known Limits

- macOS is still the primary v0.1.x target.
- Builds remain unsigned and not notarized.
- Electron itself is still a large runtime, so the app is leaner but not tiny.
- Full streaming usage extraction is still limited by provider response behavior.

## Roadmap

- Continue trimming packaged artifacts where safe.
- Add signed / notarized macOS distribution when release operations are ready.
- Add more real-world integration recipes.
- Continue improving the first-run path and demo materials.
