# Token Shredder v0.1.11

A tiny local desktop pet that shreds your AI token spending in real time.

This update focuses on first-run clarity. The backstage now has a smart next-step card that looks at the local collector, pricing, provider fields, proxy state, and usage history, then tells a new user what to do next.

## Highlights

- Added a `下一步建议` backstage card.
- Added shared setup-readiness logic for:
  - local collector status
  - editable pricing readiness
  - provider API Key / Base URL / model fields
  - local proxy enabled state
  - demo usage vs real usage
- The new card can run the no-key demo, collector test, or local proxy test directly.
- Diagnostics now include setup-readiness status, completed checks, real-usage state, and missing provider fields.
- Added unit tests for setup-readiness behavior.
- Updated README, Chinese docs, release guide, roadmap, and launch page copy.

## How To Try It

1. Open Token Shredder.
2. Right-click the desktop pet and choose `进入后台`.
3. Read the `下一步建议` card.
4. Click the suggested button:
   - `一键试玩` if this is your first time.
   - `测试本机代理` if provider fields are already configured.
   - `复制接入示例` if your script or agent can POST usage directly.

## Privacy Notes

- The setup-readiness logic only reads local app state.
- Diagnostics still redact API keys.
- No prompt, completion, message, or Authorization header is added to diagnostics.
- No cloud backend or analytics were added.

## Known Limits

- macOS remains the primary target.
- Builds are unsigned and not notarized.
- The local proxy is still basic and non-streaming usage extraction is the reliable path.
- Windows and Linux still need community testing.
