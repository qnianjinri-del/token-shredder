# Token Shredder v0.1.9

A tiny local desktop pet that shreds your AI token spending in real time.

This update continues the automation work: the packaged macOS app can now be smoke-tested, and GitHub Release assets can be verified automatically after upload.

## Highlights

- Added `npm run smoke:package`.
- `smoke:package` runs the same local API checks against the packaged `.app`.
- Added `npm run release:verify`.
- `release:verify` checks:
  - GitHub Release page
  - `.dmg` download
  - `.zip` download
  - blockmap assets
  - `latest-mac.yml`
- Added `npm run release:ship:mac`.
- `release:ship:mac` runs:
  - full checks
  - macOS packaging
  - packaged app smoke test
  - GitHub Release publishing
  - release asset verification
- Updated README, Chinese docs, release guide, roadmap, and launch page copy.

## How To Verify

```bash
npm run release:ship:mac
```

Or step by step:

```bash
npm run release:check:full
npm run dist:mac
npm run smoke:package
npm run release:github
npm run release:verify
```

## Privacy Notes

- Smoke tests use temporary local app data.
- Smoke tests send local usage numbers only.
- No prompt, completion, message, API key, or Authorization header is required.
- Release verification uses public GitHub download URLs.

## Known Limits

- macOS is still the primary target.
- Builds are unsigned and not notarized.
- Windows and Linux need community testing.
- The packaged smoke test validates app startup and local API behavior, not every visual animation frame.
- Full streaming usage extraction is not implemented.
