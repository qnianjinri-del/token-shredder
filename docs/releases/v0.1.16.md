# Token Shredder v0.1.16

A tiny local desktop pet that shreds your AI token spending in real time.

This update focuses on what happens when provider setup fails. Instead of showing only one generic error line, Token Shredder now classifies common local proxy test failures and gives a short, actionable checklist.

## Highlights

- Added provider troubleshooting logic for:
  - missing fields
  - API key / permission errors
  - model, deployment, endpoint, or path errors
  - rate limit / quota errors
  - request-shape errors
  - upstream 5xx errors
  - network / Base URL connectivity errors
  - successful responses that do not include usage
- Added troubleshooting cards to the provider setup panel.
- Added copyable provider troubleshooting reports that exclude API keys, Authorization headers, prompts, completions, messages, and request bodies.
- Added Chinese and English troubleshooting guides.
- Updated README, Chinese docs, roadmap, release guide, and launch page.
- Added unit tests for provider troubleshooting classification and privacy-safe reports.

## How To Try It

1. Open Token Shredder.
2. Right-click the desktop pet and choose `进入后台`.
3. Open the `接入` tab.
4. Fill provider fields and click `发送测试请求`.
5. If it fails, read the troubleshooting card or click `复制报告`.

## Privacy Notes

- Troubleshooting reports intentionally exclude API keys, Authorization headers, prompt text, completion text, messages, and request bodies.
- Provider templates and prices remain editable examples, not official live provider configuration or pricing.
- No cloud backend or analytics were added.

## Known Limits

- macOS remains the primary target.
- Builds are unsigned and not notarized.
- Troubleshooting is heuristic; provider-specific error formats can vary.
- The local proxy remains basic.
- Streaming usage extraction remains limited.
- Windows / Linux still need community testing.
