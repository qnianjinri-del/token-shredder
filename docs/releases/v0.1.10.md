# Token Shredder v0.1.10

A tiny local desktop pet that shreds your AI token spending in real time.

This update adds release integrity metadata. Every macOS release now gets a SHA256 manifest so people can verify downloaded artifacts.

## Highlights

- Added `npm run release:manifest`.
- Release manifest includes:
  - `.dmg`
  - `.zip`
  - `.dmg.blockmap`
  - `.zip.blockmap`
  - `latest-mac.yml`
- Manifest records:
  - asset name
  - local path
  - file size
  - SHA256
  - GitHub download URL
- GitHub Release publishing now uploads the manifest.
- `release:verify` checks that the manifest exists and covers all release assets.
- Shared release asset helpers between manifest, publish, and verify scripts.
- Updated README, Chinese docs, release guide, roadmap, and launch page copy.

## How To Verify

```bash
npm run release:ship:mac
```

Or step by step:

```bash
npm run release:check:full
npm run dist:mac
npm run smoke:package
npm run release:manifest
npm run release:github
npm run release:verify
```

## Privacy Notes

- Manifest generation reads release files only.
- It does not inspect local app data.
- It does not contain prompts, completions, messages, API keys, or usage logs.

## Known Limits

- macOS is still the primary target.
- Builds are unsigned and not notarized.
- Windows and Linux need community testing.
- The manifest verifies file integrity, not developer identity.
