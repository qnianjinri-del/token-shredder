# Token Shredder v0.1.6

A tiny local desktop pet that shreds your AI token spending in real time.

This update is a larger product polish pass for people who are trying Token Shredder for the first time and for people who want to help debug or improve it.

## Highlights

- Added a new-user checklist in the backstage window.
- Added config backup / restore.
- Backups intentionally exclude:
  - API keys
  - prompt text
  - completion text
  - messages
  - session usage logs
- Added a copyable diagnostics panel for GitHub issues and troubleshooting.
- Diagnostics redact API keys and avoid prompts / completions / messages.
- Unified usage integration examples across first-run, quick verification, and advanced monitoring.
- Added tests for backup helpers and diagnostics text.
- Updated README, Chinese docs, roadmap, and launch page copy.

## How To Try It

1. Open Token Shredder.
2. Right-click the pet and choose `进入后台`.
3. Use `从这里开始`.
4. Follow `新手任务清单`.
5. Export a config from `配置备份 / 恢复`.
6. Copy diagnostics from `问题诊断` if something breaks.

## Local API Example

```bash
curl -X POST http://127.0.0.1:17391/usage \
  -H "Content-Type: application/json" \
  -d '{"source":"my-agent","scenarioName":"first real shred","inputTokens":120000,"outputTokens":45000,"cachedInputTokens":30000,"reasoningTokens":8000}'
```

## Privacy Notes

- Backups do not include API keys or session usage logs.
- Diagnostics do not include API keys, prompts, completions, or messages.
- `/usage` examples send usage numbers only.
- The app remains local-first and listens on `127.0.0.1` by default.
- Prices are user-configured estimates, not official live provider pricing.

## Known Limits

- macOS is still the primary target.
- Builds are unsigned and not notarized.
- Windows and Linux need community testing.
- Full streaming usage extraction is not implemented.
- SDK / CLI wrappers are not implemented yet.
