# Token Shredder v0.1.0

A local desktop pet that shreds your AI token spending in real time.

## Highlights

- Pixel-art desktop shredder pet.
- Local usage collector and basic OpenAI-compatible `/v1` proxy on `127.0.0.1`.
- Codex local `token_count` watcher.
- Real-time cost visualization driven by actual usage events.
- Beginner setup for API Key, upstream Base URL, model / endpoint ID, and pricing.
- Custom Token Shredder usage JSON support.
- OpenAI-style `usage` payload support.
- Cached token handling that avoids double-charging `prompt_tokens`.
- Quiet first launch, opt-in demo mode, and real monitoring mode.
- Chinese backstage configuration UI.
- Editable sample pricing.
- Local-first privacy model.

## How To Try It

```bash
npm install
npm run dev:desktop
```

Build and run locally:

```bash
npm run start:desktop
```

Package macOS artifacts:

```bash
npm run package:mac
npm run dist:mac
```

v0.1.0 macOS artifacts are unsigned local builds.

First run:

1. Right-click the pet and choose `进入后台`.
2. Fill API Key, upstream Base URL, model / endpoint ID, and pricing.
3. Click `保存并启用`.
4. Point your OpenAI-compatible client to `http://127.0.0.1:<actualPort>/v1`.

Codex usage can also work automatically: if Codex writes new `token_count` events under `~/.codex/sessions` after Token Shredder starts, the pet will shred according to your configured prices.

## Local API Examples

Health check:

```bash
curl http://127.0.0.1:17391/health
```

OpenAI-compatible local proxy:

```ts
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "token-shredder-local",
  baseURL: "http://127.0.0.1:17391/v1",
});

await client.chat.completions.create({
  model: "your-model-or-endpoint-id",
  messages: [{ role: "user", content: "hello" }],
});
```

Custom usage:

```bash
curl -X POST http://127.0.0.1:17391/usage \
  -H "Content-Type: application/json" \
  -d '{"source":"my-agent","scenarioName":"repo cleanup","inputTokens":120000,"outputTokens":45000,"cachedInputTokens":30000,"reasoningTokens":8000}'
```

OpenAI-style usage:

```bash
curl -X POST http://127.0.0.1:17391/usage \
  -H "Content-Type: application/json" \
  -d '{"source":"openai-compatible-client","usage":{"prompt_tokens":120000,"completion_tokens":45000,"prompt_tokens_details":{"cached_tokens":30000},"completion_tokens_details":{"reasoning_tokens":8000}}}'
```

Clear session:

```bash
curl -X DELETE http://127.0.0.1:17391/usage
```

## Privacy Notes

- Runs locally on your machine.
- Collector and proxy listen on `127.0.0.1`.
- Codex watcher reads only local `token_count` events.
- Records token counts, source names, scenario names, timestamps, and local cost estimates.
- Does not record prompts or completions in the session log.
- Does not upload usage data.
- Does not use third-party analytics.
- API keys are only used locally for proxying and must not be written to logs.

## Known Limitations

- macOS is the primary target for v0.1.0.
- v0.1.0 builds are unsigned and not notarized.
- Windows and Linux are planned / experimental.
- Full streaming usage extraction is not implemented.
- Codex monitor is based on local token_count events, not official billing.
- SDK wrappers are not implemented.
- CLI wrapper is not implemented.
- Agent log watcher is not implemented.
- Pricing values are editable samples and user configuration, not official live provider pricing.
