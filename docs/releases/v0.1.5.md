# Token Shredder v0.1.5

A tiny local desktop pet that shreds your AI token spending in real time.

This update focuses on first-run usability. The goal is simple: a new user should open the app, understand the three possible paths, and copy the right integration snippet without reading the whole README first.

## Highlights

- Added a first-screen `从这里开始` backstage guide.
- Split the first-run path into three choices:
  - no-key local demo
  - provider API Key + local OpenAI-compatible proxy
  - direct `POST /usage` from scripts and agents
- Added one-click copy for:
  - curl usage example
  - JavaScript fetch usage example
  - Python requests usage example
  - OpenAI SDK local proxy example
  - paste-ready Agent / ChatGPT / Codex integration instructions
- Added anchors from the first screen to provider setup and pet skin selection.
- Moved integration snippets into a tested helper so actual local ports stay consistent.
- Updated Chinese docs, roadmap, and launch page copy.

## How To Try It

1. Open Token Shredder.
2. Right-click the pet and choose `进入后台`.
3. Use the first `从这里开始` panel.
4. Click `一键试玩`, or copy the integration instructions into your script / Agent.

## Local API Example

```bash
curl -X POST http://127.0.0.1:17391/usage \
  -H "Content-Type: application/json" \
  -d '{"source":"my-agent","scenarioName":"first real shred","inputTokens":120000,"outputTokens":45000,"cachedInputTokens":30000,"reasoningTokens":8000}'
```

## Privacy Notes

- The new snippets send usage numbers only.
- They do not require prompts, completions, messages, or API keys in `/usage`.
- The app remains local-first and listens on `127.0.0.1` by default.
- Prices are user-configured estimates, not official live provider pricing.

## Known Limits

- macOS is still the primary target.
- Builds are unsigned and not notarized.
- Windows and Linux need community testing.
- Full streaming usage extraction is not implemented.
- SDK / CLI wrappers are not implemented yet.
