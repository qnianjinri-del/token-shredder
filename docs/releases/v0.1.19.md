# Token Shredder v0.1.19

A tiny local desktop pet that shreds your AI token spending in real time.

This update makes real project integration easier: instead of only showing one-off POST examples, Token Shredder now includes copy-paste reporter helpers for JavaScript and Python projects.

## Highlights

- Added `examples/token-shredder-reporter.mjs`.
- Added `examples/token_shredder_reporter.py`.
- Added `docs/INTEGRATION_COOKBOOK.md`.
- Added JS / Python reporter helper snippets to the backstage integration recipes.
- Updated README, Chinese docs, roadmap, examples, and launch page with the new integration path.
- Reporter helpers extract OpenAI-style `usage` and subtract cached tokens from prompt tokens to avoid double counting.

## How to Try It

Start Token Shredder, then run:

```bash
node examples/token-shredder-reporter.mjs
python3 examples/token_shredder_reporter.py
```

For your own project, copy the helper file and call `reportOpenAIUsage(...)` or `report_openai_usage(...)` after a model response returns.

## Privacy Notes

- The reporter helpers send only token counts, source, scenario name, and optional direct cost.
- They do not send prompts, completions, messages, request bodies, API keys, or Authorization headers.
- Token Shredder remains local-first and listens on `127.0.0.1`.

## Known Limits

- These helpers are examples, not versioned SDK packages.
- Streaming responses still depend on whether your provider returns usage.
- macOS builds remain unsigned and not notarized.

## Roadmap
